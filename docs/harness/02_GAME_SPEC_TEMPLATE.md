# 02_GAME_SPEC_TEMPLATE — ゲーム契約テンプレート

ゲームごとに、次の契約を明文化する。  
契約とは、あとから実装を変えても壊してはいけない約束である。

```text

プレイヤーの目的:

失敗条件:
終了条件:
リトライ理由:
```

## 7. 画面状態契約

全ゲームは、最低限この状態を持つ。

```text
home
nameConfirm
countdown
playing
result
```

### 7.1 ホーム画面

必須。

```text
ゲームタイトル
短い説明
遊び方
スタートボタン
ゲームシェアボタン
必要なら「他のゲームで遊ぶ」リンク
```

ランキング対応ゲームでは、ホーム画面に名前入力欄を常時表示しない。  
スタート後に名前確認へ進める。

### 7.2 名前確認

```text
保存済み名前があれば確認する
保存済み名前がなければ入力させる
名前なしでは開始できない
前後の空白を削る
最大20文字
絵文字と記号は原則許可
確定後に blur() する
```

### 7.3 カウントダウン

必須。

```text
3
2
1
START
```

カウントダウン中に、正解、的、落下物、問題、ゲーム対象を先に見せない。

### 7.4 プレイ中

```text
名前入力を出さない
結果確定前にスコア送信しない
ゲームループは必要な時だけ動かす
画面外オブジェクトを増やし続けない
```

### 7.5 結果画面

必須。

```text
今回の記録
初回記録
ベスト記録
プレイ回数
自己ベスト更新の有無
もう一度遊ぶ
ホームへ戻る
結果シェア
ランキング送信状態
```

ランキング送信に失敗しても、結果画面を壊さない。  
送信中でも、ホームへ戻る導線は残す。  
送信中にボタン列を丸ごと消してはいけない。

## 8. ゲームタイプ別の見るべき点

### 8.1 タップ・反射ゲーム

例: イロアテ / あなたの1秒って / 充電ｶﾞｯ

```text
重視: 入力遅延、タップ判定、結果表示、ランキング、シェア、短時間での納得感
推奨技術: Level 0〜2、DOM中心またはCanvas 2D、TypeScriptは後からでよい
```

### 8.2 パズル・配置系

例: ベク取る / トマトク / イロナラベ / ハコダセ

```text
重視: 解ける盤面、誤タップ防止、ランダム生成の品質、タイムアタック、ヒントやペナルティ、理論値、seed対応の余地
推奨技術: Level 1〜3、Canvas 2D / DOM、将来TypeScript、将来Vitest、seed付き乱数
```

### 8.3 アクション・避け系

例: コロシャイン / キリガナイト / ナミオシ

```text
重視: 当たり判定の納得感、回避不能の排除、スワイプやタップの気持ちよさ、処理落ち対策、結果の納得感
推奨技術: Level 1〜5、Canvas 2D、必要ならMatter.js、必要ならThree.js
```

### 8.4 防衛・シューティング系

例: うちかえる / マロン飛行 / ヒトを許すな！

```text
重視: 敵の増え方、後半難易度、処理落ち、スコア上限、ランキング異常値、クリア不能状況の排除
推奨技術: Level 2〜4、敵・弾・効果をデータ化、TypeScript候補、自動テスト候補
```

### 8.5 クイズ・相談文系

例: トレメシ

```text
重視: 問題文が自然である、選択肢の意味が分かる、用語説明がある、誤解しにくい、ランキング表示が壊れない
推奨技術: Level 2〜3、データ分離、CSV/JSON検証、TypeScript候補
```

### 8.6 3D・奥行き系

例: ジョーバ / Helix Jump系の研究 / コロシャインの将来拡張

```text
重視: 奥行き、カメラ、接触判定、回転の重さ、iPhone 11 Proでも破綻しない性能
推奨技術: Level 4〜5、Three.js、WebGL、性能計測
```
