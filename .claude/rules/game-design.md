# Game Design Rules

- 小さくても雑にしない。「すぐ分かる → すぐ触れる → 失敗理由が分かる → 結果が出る → ランキングに残る → シェアできる → もう一度遊べる」の流れを守る。
- 技術よりゲーム体験を先に判断する。ゲームの核が何かを最初に言語化する。
- 参考ゲームを使う時は、まず参考元の面白さの核を理解し、成立する最小構造を作ってから、テーマ、見た目、演出、キャラクター、難易度、シェア文、ランキングをカメレオンJP向けに変える。
- 最初から無理にオリジナル化してロジックを壊さない。
- 全ゲームは home / nameConfirm / countdown / playing / result の画面状態を持つ。
- カウントダウンは 3・2・1・START。カウントダウン中にゲーム対象を先に見せない。
- 結果画面には、今回記録、初回記録、ベスト記録、プレイ回数、もう一度遊ぶ、ホームへ戻る、結果シェアを置く。
- ランダム生成は seed（再現用の値）で再現できる形にする。Math.random() の直接使用は避ける。
- クリア不能な盤面、序盤の理不尽配置、急すぎる難易度上昇を出さない。
- ランキング上位が運だけで決まらないようにする。
- 同じ処理を3ゲーム以上で使ったら共通化候補にする。ただし、ゲームのテンポ、結果コメント、演出の個性、キャラクター性、難易度の味付けは共通化しない。
- 共通化は事故を減らすために行う。ゲームの個性を消すためではない。
